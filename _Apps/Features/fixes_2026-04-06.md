# LanobeReader 不具合修正まとめ (2026-04-06 〜 2026-04-07)

このドキュメントは、初版実装後に判明した不具合と、その修正内容をまとめたもの。

---

## 1. Android スレッド違反 (`Only the original thread that created a view hierarchy can touch its views`)

### 原因
ViewModel 内で `await ...ConfigureAwait(false)` を多用しており、`await` 以降の処理がスレッドプール（Thread-4 等）で実行されていた。その状態で `ObservableProperty` のセッターを呼ぶと、MVVM Toolkit の PropertyChanged 通知がバックグラウンドスレッドから UI バインディングを更新し、Android のスレッド安全性チェックに違反していた。

### 修正
全 ViewModel から `.ConfigureAwait(false)` を削除。MAUI ViewModel は UI 層なので `SynchronizationContext` を維持して UI スレッドに戻すべき。
不要になった `MainThread.BeginInvokeOnMainThread` / `InvokeOnMainThreadAsync` ラッパーも除去。

### 修正対象
- `_Apps/ViewModels/SearchViewModel.cs`
- `_Apps/ViewModels/NovelListViewModel.cs`
- `_Apps/ViewModels/ReaderViewModel.cs`
- `_Apps/ViewModels/EpisodeListViewModel.cs`
- `_Apps/ViewModels/SettingsViewModel.cs`

## 2. なろう エピソード一覧が「本編」1話だけになる

### 原因
なろうのサイトリニューアルで HTML の CSS クラス名が刷新されており、旧セレクター (`.index_box`, `.chapter_title`, `.novel_sublist2`) で要素が取れず、短編フォールバックが走っていた。
また 100 話ごとのページ分割 (`?p=2`, `?p=3`...) にも未対応だった。

### 修正
`_Apps/Services/Narou/NarouApiService.cs` の `FetchEpisodeListAsync` を新セレクターに更新し、ページネーション対応を追加。

| 用途 | 旧セレクター | 新セレクター |
|---|---|---|
| 目次コンテナ | `.index_box` | `.p-eplist` |
| 章タイトル | `.chapter_title` | `.p-eplist__chapter-title` |
| エピソード行 | `.novel_sublist2` | `.p-eplist__sublist` |
| エピソードリンク | `a` | `.p-eplist__subtitle` |

ページネーションは `.c-pager__item--next` が `<a>` であるかぎり次ページを取得。

## 3. なろう 本文に HTML タグ (`<p id="L1">` 等) が残る

### 原因
旧コードは `InnerHtml` に対し `Replace("<p>", "")` 等の文字列置換で HTML を剥がしていたが、新サイトの段落タグは `<p id="L1">` のように属性付きのため剥がれず、そのまま画面に表示されていた。

### 修正
`NarouApiService.FetchEpisodeContentAsync` で本文セレクターを更新し、AngleSharp で段落要素を取得して `TextContent` を結合する方式に変更。

| 用途 | 旧 | 新 |
|---|---|---|
| 本文セレクター | `#novel_honbun` | `.js-novel-text.p-novel__text:not(.p-novel__text--afterword)` |
| テキスト抽出 | `InnerHtml` + 文字列置換 | `QuerySelectorAll("p").Select(p => p.TextContent)` |

カクヨム側 `KakuyomuApiService.FetchEpisodeContentAsync` も同様に修正。

> ⚠️ アプリ DB にキャッシュ済みの古い本文は自動更新されない。一覧で「キャッシュ削除」してから再表示する必要がある。

---

## 4. なろう「完結」/「連載中」の判定が逆

### 原因
なろう API の `end` フィールドの仕様が直感に反しており、**`end=1` が連載中、`end=0` が完結済み**。
旧コードは `end == 1` を完結と判定していた。

### 修正
`NarouApiService` の `SearchAsync` / `FetchNovelInfoAsync` で判定を `end == 0` に反転。

> ⚠️ DB に保存済みの `IsCompleted` は自動上書きされない。再登録 or 更新チェック経由で `FetchNovelInfoAsync` が走った時点で正しくなる。

---

## 5. リーダー画面でスクロールできない

### 原因
親 `Grid` に `TapGestureRecognizer`（ヘッダー/フッター表示切替用）を付けており、それが `ScrollView` のタッチイベントを奪っていた。
試行錯誤の過程で `Label` に移してもスクロールを阻害したため、最終的にトグル機能は廃止。

### 修正
`_Apps/Views/ReaderPage.xaml`:
- 親 `Grid` の `TapGestureRecognizer` を削除
- `ScrollView` 上の `SwipeGestureRecognizer`（左右スワイプでページ送り）も削除（スクロールと競合する可能性あり）
- ヘッダー/フッターは常時表示

`_Apps/Views/ReaderPage.xaml.cs`:
- `OnSwipedLeft` / `OnSwipedRight` ハンドラーを削除
- 末尾までスクロールしたら既読化する `OnScrolled` のみ残す

ページ送りはフッターの「◀ 前へ」「次へ ▶」ボタンで操作。

## 6. 一覧の長押しメニューが Android で動かない

### 原因
`FlyoutBase.ContextFlyout` / `MenuFlyout` は Windows / macOS 専用で **Android 非対応**。

### 修正
`_Apps/Views/NovelListPage.xaml` で `MenuFlyout` を削除し、`SwipeView` (Android で自然な左スワイプメニュー) に置き換え。

- 左スワイプ → 「キャッシュ削除」(オレンジ) と「削除」(赤) のボタンが出現
- タップは従来どおり詳細画面への遷移

## 7. カクヨム検索結果が出ない

### 原因
カクヨムのサイトが React/Next.js 化されており、旧セレクター (`.widget-workCard` 等) は存在しない。

### 修正
`_Apps/Services/Kakuyomu/KakuyomuApiService.cs` の `SearchAsync` を新構造に対応。
`<a title="..." href="https://kakuyomu.jp/works/{id}" class="...">` パターンを抽出し、`title` 属性からタイトルを取得。重複は `HashSet<string>` で除外。

## 8. カクヨム エピソード一覧が 0 話になる / 全話取得できない

### 原因
カクヨムは React/Next.js による SPA 化と同時に、目次がアコーディオン折りたたみ実装になり、`<a href="/episodes/...">` の DOM 要素は最初の 30 話程度しか HTML に出力されていない。残りは JS 実行で動的にレンダリングされる。

### 解決方針
ブラウザを起動せずに全話取得するため、SSR で埋め込まれている **`__NEXT_DATA__` 内の Apollo State** を直接パースすることにした。Apollo State には全エピソード情報が含まれている。

### Apollo State の構造
```
props.pageProps.__APOLLO_STATE__
├─ "Work:{novelId}"                  → serialStatus ("RUNNING" / "COMPLETED")
├─ "TableOfContentsChapter:{id}"     → episodeUnions[] (順序保持)
│    └─ {"__ref": "Episode:{episodeId}"}
└─ "Episode:{episodeId}"             → title, id, publishedAt
```

### 実装
`KakuyomuApiService.cs` に以下のヘルパーを追加:
- `ExtractApolloState(html)` — `<script id="__NEXT_DATA__" type="application/json">` から JSON を切り出し、`props.pageProps.__APOLLO_STATE__` を `JsonElement` として返す
- `ParseEpisodesFromApolloState(html)` — `TableOfContentsChapter:*` を順番に走査し、各章の `episodeUnions` を辿って `Episode` 型のリストを構築（章名 `ChapterName` も保持）
- `ParseEpisodeIdsFromApolloState(html)` — エピソード ID のみを順番付きで返す（本文取得時の URL 構築に使う）

公開メソッドの更新:
- `FetchEpisodeListAsync` → `ParseEpisodesFromApolloState` を使用、全話 + 章名を取得
- `FetchEpisodeContentAsync` → `ParseEpisodeIdsFromApolloState` でエピソード ID を取得し、`/works/{novelId}/episodes/{episodeId}` を直接構築
- `FetchNovelInfoAsync` → `Work:{novelId}` の `serialStatus == "COMPLETED"` で完結判定、エピソード総数も Apollo State 由来

### 補足
- 本文ページ (`/works/.../episodes/...`) 自体は旧構造の `.widget-episodeBody` がそのまま使えており、ここはセレクター変更不要だった
- 段落抽出は `<p>` の `TextContent` を結合（なろうと同じ方針）
