# 一括選択・一括操作（F-015）実装プラン

> このプランは別セッションが追加調査なしで実装着手できる粒度で記述する。要件は `新刊チェッカー_要件定義書.md` §3.2.8 / F-015、規約は `_Apps/CLAUDE.md` を正とする。

## 1. 確定仕様（決定事項）

| 項目 | 決定 |
|---|---|
| 対象画面 | シリーズ詳細(SCR-005) / お気に入り(SCR-007) / 発売予定(SCR-008) / ランキング(SCR-009) |
| アクション | 005=購入済/解除、007=購入済/解除＋お気に入り解除、008/009=お気に入り/解除 |
| 選択モード入口 | ヘッダ ToolbarItem「選択」（長押しは不採用＝MAUI標準に無い） |
| 操作UI | 画面下部のアクションバー（選択件数・全選択・各アクション・完了） |
| 行タップ | 選択モード中=選択トグル / 通常時=巻詳細へ遷移 |
| 戻るボタン | 選択モード中は解除（画面離脱しない） |
| 非永続巻 | 一括お気に入り時に `SeriesId=NULL` で INSERT（共通サービスに集約） |
| 孤児行 | 方式C：新刊チェック開始時に一括掃除（即時削除しない） |
| 選択状態の保持 | 方式B：`BookListItem` を Observable 化し `IsSelected` を持たせる（CollectionView標準のSelectionMode/SelectedItemsは使わない＝既存の行ジェスチャ→VMコマンド方式と競合するため） |

## 2. 設計方針

### 2.1 方式B採用の理由（CollectionView標準選択=方式Aを採らない）
- 全一覧が `TapGestureRecognizer` → `Command="{Binding BindingContext.XxxCommand, Source={x:Reference Root}}"` でVMコマンドに流す作りで統一されている（`x:DataType` 不採用もこのため）。
- 方式A（`SelectionMode="Multiple"`＋`SelectedItems`）は、この行内ジェスチャと標準選択がタップを取り合って競合する。回避には行ジェスチャ撤去＋`SelectionChangedCommand`化が必要で、かえって既存構造から外れる。
- 方式Bは「選択モード中はタップで `IsSelected` をトグル、通常時はナビゲート」と既存コマンド経路を流用でき、チェックボックス表示も明示的。

### 2.2 共通基底 `SelectableBookListViewModel`
選択モードの定型処理（モード開始/終了・トグル・全選択・件数・一括適用）を基底に集約する。差分（対象コレクション・永続Bookへの解決・再読込・遷移）は抽象フックで各VMが提供。

- 既存の `ApiBrowseViewModel`（Ranking/Upcoming の基底）・`SeriesDetailViewModel`・`FavoritesViewModel` をこの基底の派生に載せ替える。

### 2.3 非永続巻の永続化を Core 共通サービスへ
`BookDetailViewModel.EnsurePersistedAsync`（[BookDetailViewModel.cs:135](NewReleaseChecker.App/ViewModels/BookDetailViewModel.cs#L135)）にしか無い INSERT-on-demand を Core の `BookActionService.EnsurePersistedAsync(RakutenBook)` に抽出し、一括処理と巻詳細で共有する（コピペ回避）。

### 3.1 Core 層

#### 新規 `NewReleaseChecker.Core/Services/BookActionService.cs`
```csharp
using NewReleaseChecker.Core.Abstractions;
using NewReleaseChecker.Core.Models;

namespace NewReleaseChecker.Core.Services;

/// <summary>
/// 巻のフラグ操作に伴う共通ロジック。非永続巻（RakutenBook）の SeriesId=NULL 永続化を集約する。
/// </summary>
public sealed class BookActionService
{
    private readonly IBookRepository _book;
    public BookActionService(IBookRepository book) => _book = book;

    /// <summary>
    /// 非永続巻を SeriesId=NULL で永続化して返す。既存（ItemNumber 一致）があればそれを返す。
    /// 裏のチェックと同 ItemNumber を同時 INSERT して UNIQUE に抵触した場合は、読み直して既存行を返す。
    /// </summary>
    public async Task<Book> EnsurePersistedAsync(RakutenBook src)
    {
        var existing = await _book.GetByItemNumberAsync(src.ItemNumber);
        if (existing is not null) return existing;

        var b = new Book
        {
            SeriesId = null,
            ItemNumber = src.ItemNumber,
            Isbn = src.Isbn,
            Title = src.Title,
            Author = src.Author,
            Publisher = src.Publisher,
            ReleaseDate = ReleaseDateParser.ToIso(src.SalesDate),
            ImageUrl = src.ImageUrl,
            ItemUrl = src.ItemUrl,
            Caption = src.Caption,
            DetectedAt = DateTime.Now.ToString("yyyy-MM-ddTHH:mm:ss"),
        };
        try
        {
            b.Id = await _book.InsertAsync(b);
            return b;
        }
        catch
        {
            // UNIQUE 競合等。読み直して既存があればそれを採用、無ければ再送出。
            var raced = await _book.GetByItemNumberAsync(src.ItemNumber);
            if (raced is not null) return raced;
            throw;
        }
    }
}
```
- 任意（推奨）: `BookDetailViewModel` の source バインド時の永続化もこの `EnsurePersistedAsync(_source)` を使うようリファクタし、ロジックの二重管理を解消する（挙動同一）。

#### 新規 `NewReleaseChecker.App/ViewModels/SelectableBookListViewModel.cs`
```csharp
using System.Collections.ObjectModel;
using CommunityToolkit.Mvvm.ComponentModel;
using CommunityToolkit.Mvvm.Input;
using NewReleaseChecker.App.Models;
using NewReleaseChecker.Core.Abstractions;
using NewReleaseChecker.Core.Models;
using TBird.Core;

namespace NewReleaseChecker.App.ViewModels;

/// <summary>巻一覧の一括選択・一括操作の共通基底（F-015）。</summary>
public abstract partial class SelectableBookListViewModel : ObservableObject
{
    protected readonly IBookRepository BookRepo;
    protected SelectableBookListViewModel(IBookRepository bookRepo) => BookRepo = bookRepo;

    [ObservableProperty] private bool _isSelectionMode;
    [ObservableProperty] private int _selectedCount;

    /// <summary>選択対象のコレクション（各VMの Books / Items を返す）。</summary>
    protected abstract ObservableCollection<BookListItem> SelectionItems { get; }

    /// <summary>行アイテムを永続 Book に解決する（永続=BookId取得 / 非永続=EnsurePersist）。null は対象外。</summary>
    protected abstract Task<Book?> ResolveAsync(BookListItem item);

    /// <summary>一括適用後の再読込（不要な画面は Task.CompletedTask）。</summary>
    protected abstract Task ReloadAsync();

    /// <summary>通常タップ時の遷移（各VMの巻詳細遷移）。</summary>
    protected abstract Task OpenBookAsync(BookListItem item);

    [RelayCommand] private void EnterSelectionMode() => IsSelectionMode = true;

    [RelayCommand]
    private void ExitSelectionMode()
    {
        foreach (var it in SelectionItems) it.IsSelected = false;
        SelectedCount = 0;
        IsSelectionMode = false;
    }

    [RelayCommand]
    private void RowTapped(BookListItem? item)
    {
        if (item is null) return;
        if (IsSelectionMode)
        {
            item.IsSelected = !item.IsSelected;
            SelectedCount = SelectionItems.Count(i => i.IsSelected);
        }
        else _ = OpenBookAsync(item);
    }

    [RelayCommand]
    private void SelectAll()
    {
        var allSelected = SelectionItems.Count > 0 && SelectionItems.All(i => i.IsSelected);
        foreach (var it in SelectionItems) it.IsSelected = !allSelected;
        SelectedCount = SelectionItems.Count(i => i.IsSelected);
    }

    /// <summary>選択巻に mutate を適用：解決(EnsurePersist含む)→mutate→UpdateFlags→再読込→モード解除。</summary>
    protected async Task ApplyToSelectedAsync(Action<Book> mutate)
    {
        var targets = SelectionItems.Where(i => i.IsSelected).ToList();
        if (targets.Count == 0) { ExitSelectionMode(); return; }
        try
        {
            foreach (var item in targets)
            {
                var b = await ResolveAsync(item);
                if (b is null) continue;
                mutate(b);
                await BookRepo.UpdateFlagsAsync(b);
            }
        }
        catch (Exception ex) { MessageService.Exception(ex); }
        finally
        {
            ExitSelectionMode();
            await ReloadAsync();
        }
    }
}
```

## 5. 検証（実機/エミュレータ）
1. シリーズ詳細：選択→複数購入済→反映・バッジ更新／購入済解除。
2. お気に入り：複数お気に入り解除→一覧から消える／追跡巻は SeriesId 保持（シリーズ詳細には残る）。
3. ランキング：複数お気に入り→お気に入り一覧に「未追跡」で出現。再度ランキングで同巻を解除→お気に入りから消える。
4. ランキングで一括お気に入り→解除した巻が、次回チェック開始時に孤児掃除で消える（DB件数確認）。
5. 選択モード中の戻るボタンでモード解除（画面は離脱しない）。
6. 通常タップは従来どおり巻詳細へ遷移。

## 6. リスクと対応
- **UNIQUE競合**: `EnsurePersistedAsync` の try/catch＋読み直しで吸収（§3.1）。
- **孤児行蓄積**: 方式C（チェック開始時掃除）。即時削除しないため単体操作の挙動は不変。
- **チェックボックス可視性**: `Source={x:Reference Root}` でページVMの `IsSelectionMode` を参照（既存のコマンド参照と同方式）。`x:DataType` 非採用方針を維持。
- **タブ root ページの戻るボタン**: `OnBackButtonPressed` override で選択モードを優先解除。
- **ReloadAsync の範囲**: 永続巻VMは再読込で表示更新、API一覧VMは表示変化が無いため no-op。

## 7. 対象外（今回スコープ外）
- シリーズ一覧（SCR-002/003。巻一覧ではなくシリーズ一覧のため）。
- 長押しジェスチャ（将来 `CommunityToolkit.Maui` の `TouchBehavior.LongPressCommand` で「追加の入口」として拡張可能）。
- ランキング/発売予定の一括「購入済」（必要になれば ApiBrowse にコマンド追加で対応可）。
